CRM API Documentation

Welcome to the CRM API documentation. This document provides all the information you need to interact with our CRM API.

Overview

The API allows you to perform operations such as retrieving data and submitting requests over Curtomers database. Below is a list of available endpoints and their usage.

Endpoints

GET /api/v1/customers

Retrieve a list of customers.

Example

Request


    curl --location 'localhost:3000/api/v1/customers'

Response


    {
        "1": {
            "ID": "1",
            "Name": "Alice",
            "Role": "Software Engineer",
            "Email": "alice@example.com",
            "Phone": "6785556789",
            "Contacted": false
        },
        "2": {
            "ID": "2",
            "Name": "Bob",
            "Role": "Account Manager",
            "Email": "bob@example.com",
            "Phone": "6785556780",
            "Contacted": true
        }
    }

POST /api/v1/customers

Create a new customer.

Example

Request


    curl --location 'localhost:3000/api/v1/customers' \
    --header 'Content-Type: application/json' \
    --data-raw '{
        "ID": "4",
        "Name": "Alanis",
        "Role": "Software Engineer",
        "Email": "alanis@example.com",
        "Phone": "6785556783",

GET /api/v1/customers/{ID}

Retrieve a specific customer by ID.

Example

Request


    curl --location 'localhost:3000/api/v1/customers/1'

Response


    {
        "ID": "1",
        "Name": "Alice",
        "Role": "Software Engineer",
        "Email": "alice@example.com",
        "Phone": "6785556789",
        "Contacted": false
    }

PUT /api/v1/resources/{ID}

Update a specific customer by ID.

Example

Request


    curl --location --request PUT 'localhost:3000/api/v1/customers/4' \
    --header 'Content-Type: application/json' \
    --data-raw '{
        "ID": "4",
        "Name": "Alanis",
        "Role": "Technical Leader",
        "Email": "alanis@example.com",
        "Phone": "6785556783",
        "Contacted": false
    }'

Response


    {
        "ID": "4",
        "Name": "Alanis",
        "Role": "Technical Leader",
        "Email": "alanis@example.com",
        "Phone": "6785556783",
        "Contacted": false
    }

PATCH /api/v1/resources

Update customers in Batch.

Example

Request


    curl --location --request PATCH 'localhost:3000/api/v1/customers' \

            "ID": "1",
            "Name": "Alice Wonderful",
            "Role": "Software Engineer",
            "Email": "alice@example.com",
            "Phone": "6785556789",
            "Contacted": false

        {
            "ID": "2",
            "Name": "Bob Ferguson",
            "Role": "Account Manager",

            "Phone": "6785556780",
            "Contacted": true

Response
